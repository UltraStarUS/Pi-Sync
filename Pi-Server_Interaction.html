Basic Explanation

    There are three ways that you can interact with the Pi Server remotely.

  1. The SMB / Samba Share
  2. VNC (Virtual Network Computing)
  3. SSH (Secure Shell)

The following scripts and process will be utilizing the last option, ssh, in order to connect the end-user system (you) to the server.

Changing Permissions of Files / Directories and Moving Files around using CLI

This is more `Linux 101` than it is specific information for the Pi Server, but it is still important nonetheless. 

You have 5 commands for moving files around, and 2 commands for changing data permissions

Moving: mv, cp, rm, scp, mount. Permissions: chmod, chown

I highly suggest that you copy data, especially that which is important, rather than move it. It is essentially the difference between copy and cut. I've been burned by the mv (move) command before, I hope you don't too. 

Move or Copy Data from Point A to Point B

  1. First, locate Point A and Point B. This can be directories or files, you need to know the path of the data before you can move it.
  2. Second, confirm that your user has the permissions to move a file and that it will not disrupt a program by moving or copying it.
  3. Syntax: for cp, mv, scp, and mount you execute the command essentially the same. 
    1. `[program] -[modifiers] path/to/source path/to/destination
  4. However, be careful as to what you mv as it can cause serious damage pulling configuration files around on a running server.
  5. A Key Trick with cp, mv and scp is that you can overwrite a file by moving one into another, or copying it into another file. 
  6. Another is to use the `*` character to delineate between the folder and the folder's contents
    1. Example 1: `cp -r /home/pi/Documents/Testfolder/ ~/Testfolder1/`
    2. Example 2: `cp -r /home/pi/Documents/Testfolder/* ~/Testfolder1/`
    3. 1 will copy the folder `Testfolder` and its content, while 2 will copy only the contents. Both will output to the same folder however.

Mounting

There are limited times you will actually need to mount anything to the Server, so I will show one instance. 

If the Vista Server becomes disconnected, or stops syncing with the Pi Server then you will most likely need to execute the following: 

sudo mount -t cifs '//vista-akchin/VISTA/Extracts/ExtractResults/' /home/pi/.webbackup/mntpoint/ -o cred=/root/.cred/.cred

The mount command commonly requires sudo / root level access in order perform its actions as a result of them being system / environment changes. However, like most commands dealing with data it follows a similar syntax to the move and copy commands (mount -[modifiers] path/to/source path/to/destination -[modifiers]).

This will also be in a script under the `~/.script/` directory, the `mount.sh` script will automate the specific command above.

Remove Data 

  1. The blessed rm command, is massively cursed.
  2. Take a moment before you  hit enter whenever you remove a file or directory; as it is almost always unrecoverable. 
  3. Follow steps 1 and 2 from Move or Copy, replacing moving and copying with removing and deleting.
  4. Syntax: rm -[modifiers] path/to/file/or/directory
  5. When deleting a folder, and everything in it you can use the `r` and `f` modifiers 
    1. Example: `rm -rf /home/pi/Documents/Testfolder/`
  6. As with Move or Copy, you can use the `*` character to delineate between the folder and the folder's contents
    1. Example 1: `rm -rf /home/pi/Documents/Testfolder/`
    2. Example 2: `rm -rf /home/pi/Documents/Testfolder/*`
    3. 1 will delete the folder `Testfolder` and its content, while 2 will delete only the contents and leave `Testfolder` up

Changing file and directory permissions / Ownership 

The two commands, chown and chmod conduct changes to ownership and permissions respectively. 

  1. These commands are commonly run with sudo/root level access, so prepare accordingly
    1. Executing `sudo -su` and typing in the passwod for the account will give you that access.
  2. Follow steps 1 and 2 from Move or Copy, replacing moving and copying with changing ownership and permissions.
  3. Syntax: chown [owner]:[group] /path/to/file/or/directory OR chmod [permission] /path/to/file/or/directory
    1. Owner being the user that will become the owner in chown
  4. Using these commands in combination with each other allows you to limit read, write, and exectue permissions on any given file or directory (even some you shouldn't touch) to specific users and groups.
  5. Using a website like Chmod Calculator allow you to better modify permissions. 
    1. Once you decide the permissions you want to grant or deny you plug in the number to the `permission` portion from step 3.
  6. A few common actions are:
    1. sudo chown www-data:www-data filename
    2. sudo chmod 700 filename
    3. sudo chmod 777 filename
    4. sudo chmod 660 ~/.ssh/

Use these commands wisely.
