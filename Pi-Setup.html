Hi! So this is going to be the quick guide to getting a new pi up and running. I want to get this out quickly but I will update it with pictures soon!

Materials

  • Your Desktop
  • A pi
  • The scripts from the piserver (\\piserver\Scripts)
  • A new Image (\\piserver\IMG)
  • Balenaetcher installed on your system (from here)
  • Some way to read and write data on a microSD card

Procedure

  1. Grab the Image from the piserver 
    1. You can download it from \\piserver\IMG in the Windows File Explorer
  2. Get the microSD card out of the Pi you are setting up and plug it into your system using an adapter
  3. Open balenaetcher or get it downloaded here
  4. For `Select Image` choose the image that you just downloaded from the piserver
  5. Confirm that the MicroSD card is the selected target, then hit `Flash!`
  6. Once it has completed, put the MicroSD back into the Pi and get it setup at the refrigerator
    1. There are a set of 3 cables connected with a velcro loop. Plug in all of them in any order except power, which should be last. (Ethernet, HDMI, Power)
  7. Once it begins to Power on it will go through a long boot process, and may reboot as it starts. This is to be expected and may take some time.  
  8. Get the Scripts folder open on your desktop
    1. Downloaded from \\piserver\Scripts in the Windows File Explorer
  9. Run `PiServer_Connect.bat`
    1. If it asks you yes or no, just type yes. It should connect quickly 
    2. If Smartscreen pops up, you can press yes.
    3. Continue when the following window pops up
  10. Once the Pi displays a desktop on the monitor, run the command `testpi`. This connects you to the Pi. 
    1. If this doesn't work, wait for a little bit; it can sometimes take a few minutes for the hostname to propagate to the PiServer
  11. Now, run `config`
  12. The script will first ask for a hostname, use the naming structure found here
  13. It will then come to a screen similar to the one pictured below. Use the steps outlined hereto edit the file, when you've made the necessary changes press `ctrl+o`, `enter`, then `ctrl+x`
    1. NOTE: If the pi was a chrome display board and you are reconfiguring it, you have to delete the line at the bottom that has `autochrome` in it.
  14. It will then ask you if the Pi is WiFi, Ethernet, or a Chrome Display Board. 
    1. WiFi means the Pi will be connected to the UltraSecure WiFi network, 
    2. Ethernet means it will be connected to the network via an Ethernet cable, 
    3. Chrome Display Board means any of the Theaterboards, and most of the Box Office signage (as well as bowling1)
      1. Refer to this to determine if it is a Chrome Display Board
      2. If it is a Display board it will ask again if it is WiFi or Ethernet, the same applies here as it does at step 14.
  15. It will then ask you what you want to set as the source folder, from the selection choose whichever folder the Pi is meant to use. Do not type the `./` or the `:`.
    2. An example input would be one of the following:
    3. Main.Concessions/Combo.Boards
    4. Harrahs.Bridge/Theater
    5. JBar
  16. Finally, it will begin syncing for the first time. Posterboards tend to take longer due to the number and size of files and will be the bulk of the time spent on the setup process. 
    1. However, if it fails such as the picture below, wait for the pi to restart, then open a terminal window or ssh back into it, and execute the `cleaner` command.
  17. Once the Pi reboots it will be ready to deploy or redeploy
  18. Nice work! You did it!

Issues and Changes

Use the account `pi` and the password `ULTRAPi`. 

You should be able to use either the hostname of the pi or it's IP address to connect in to it. Except WiFi, that is IP address connections only.

You can reconfigure a pi if it is deployed using the newest images regardless of where it is or what it's doing. 

You can check the file `Pi_Guest-List.txt` on the Pi Server to see which pis are syncing and when, as well as how they are doing so (hostname or IP address). 

Pis that utilize the chrome displayboard feature will not sync, nor ask to be synced.
